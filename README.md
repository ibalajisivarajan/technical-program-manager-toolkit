## 🚀 How To Use

**Option 1 — Use directly on GitHub**
Click any template link in the table above → view it directly in your browser → copy the content into your own docs.

**Option 2 — Clone the repo**
```bash
git clone https://github.com/ibalajisivarajan/technical-program-manager-toolkit.git
cd technical-program-manager-toolkit/templates
```

**Option 3 — Download individual templates**
Click any template → click the **Raw** button → Save As to download.

## 📋 Template Previews

### 📋 Program Charter
Defines the "why, what, and who" of a program before a single line of code is written.

```
Program Name · Sponsor · Objectives · Scope In/Out
Success Metrics · Key Stakeholders · Timeline · Risks
Dependencies · Budget · Approval Sign-off
```

---

### 🔗 Dependency Tracker
Maps every cross-team dependency so nothing falls through the cracks.

```
Dependency · Owned By · Needed By · Status · Blocker?
Due Date · Escalation Path · Resolution Notes
```

---

### 🏛️ Architecture Review
Structured checklist for reviewing system design decisions at scale.

```
Problem Statement · Proposed Solution · Alternatives Considered
Trade-offs · Non-Functional Requirements · Security Review
Data Flow · Rollback Plan · Decision Record
```

---

### ⚠️ Risk Register
Living document for tracking program risks, owners, and mitigation actions.

```
Risk ID · Description · Likelihood · Impact · Score
Owner · Mitigation Plan · Contingency · Status · Last Updated
```

---

### 🚀 Release Orchestration
Coordinates multi-service releases with clear go/no-go gates.

```
Release Name · Services In Scope · Release Owner
Pre-release Checklist · Go/No-Go Criteria · Rollback Plan
Communication Plan · Post-release Validation · Retrospective
```
